# Systems-lab

A personal learning laboratory for mastering **systems programming, data engineering, and modern backend development**. 

This repo is my dedicated space to build strong, production-grade skills through hands-on projects — starting with TypeScript & Node.js, and eventually expanding into Go, Python, C++, and other languages.

### Goals
- Deepen my understanding of systems concepts (streaming, concurrency, I/O efficiency, pipelines, observability, etc.)

---

## 25 Projects – TypeScript + Node.js (70% Systems + 30% Web)

### Beginner

- [ ] 1. **CLI File Analyzer** — Robust CLI tool for analyzing JSON/CSV files (schema, stats, null counts, type detection)

- [ ] 2. **Streaming File Processor** — Process large files using Node.js Streams without loading them fully into memory

- [ ] 3. **Simple Hono API** — Basic REST API using Hono with TypeScript and Zod validation

- [ ] 4. **File Chunker Utility** — Split, process, and recombine large files efficiently

- [ ] 5. **Local Key-Value Store** — Persistent flat-file based key-value store with indexing

### Intermediate

- [ ] 7. **Hono File Upload API** — API for handling single and multiple file uploads

- [ ] 8. **Concurrent Downloader** — Parallel file downloader using worker_threads

- [ ] 9. **Streaming Log Analyzer** — Real-time/batch log processing and pattern detection

- [ ] 10. **Basic Job Queue System** — Job orchestration using BullMQ with retries and logging
